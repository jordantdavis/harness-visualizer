cchv — interactive TUI redesign (two-column slide model)

Two-column master-detail (≥80 cols). Browse shows Sessions │ Events; pressing Enter on an event slides into Drill = Events │ Inspector (Sessions hides, Inspector reveals). Esc/h pops one step shallower — never quits. Events is the fixed pivot: it stays under your eye and slides left as the panes reflow, so nothing teleports. Selection = caret gutter + accent bar + subtle band (never reverse-video); columns are fixed-width. 16-color ANSI tokens from theme.go; color always redundant with glyph + position. Overflow is softened: type-aware truncation keeps the meaningful part (paths keep the filename), a dim … flags a clipped value, and the Inspector soft-wraps instead of clipping.

Navigation ladder (the slide model): Sessions │ Events (Browse) ⟷ Events │ Inspector (Drill). Enter/l on an event enters Drill (Enter on a session just focuses Events — Sessions stays, so you can hop between concurrent same-cwd sessions). Esc/h always pops exactly one step shallower: Inspector → Events (back to Browse) → Sessions → stop. Esc never quits — only q quits. Tab cycles the visible panes; 1/2/3 jump to Sessions/Events/Inspector and cross the depth boundary when needed (3 from Browse enters Drill; 1 from Drill pops out). Events is the pivot — it never teleports; Sessions slides out left while Events takes its slot (instant swap under reduced-motion).
Selection vs focus: the selected row always shows the caret + accent bar. When the pane itself is focused, the band brightens (--sel-bg-foc) and the caret/accent use the info accent; when the pane is unfocused the same row keeps a dimmer band (--sel-bg) with a grey caret — so you can always see "where you'll land when you tab here" without a second jarring highlight.
Error rows use the red accent bar + ✘ glyph + red (exit 2) text — three redundant signals.
Folded Pre/Post ops render as one row labeled PreToolUse·Post with the resolved glyph and the Post−Pre duration; an unpaired Pre stays ▶ running until its Post lands.
Stable columns: TIME(8) · ST(1+pad) · HOOK(16) · TOOL(12) · TARGET(flex) · DUR(right-aligned 7), with a 1-char caret gutter to the left of TIME on every row including the header. Two columns give TARGET much more room — full paths in Browse, gracefully ellipsized in Drill where the Inspector takes width.
Soft truncation (type-aware): the list clips but keeps the signal, flagged by a dim … in the last cell. Paths → basename-priority middle-elision internal/…/middleware.go then …/middleware_test.go (the filename is sacred). Commands → head-keep go test ./internal/au…. Quoted strings → head-keep, marker after "func.*Handler" inter…. The marker eats one cell (content clipped to width−1) so "clipped" is a deliberate flag, and its position tells you which end was dropped (leading/interior … = path, trailing … = head-keep). Under NO_COLOR the … stays a literal glyph (or ~ if unsupported) — color-blind / dumb-term safe.
Wrap vs clip: the list never wraps (fixed-width, no reflow — including the selected row, whose full TARGET is shown in the Inspector key-facts instead of a jumpy inline reveal). The Inspector soft-wraps long values with a hanging indent aligned under the value start — it's the full-data escape hatch and loses nothing. A dim … is an affordance: Enter to inspect the full value, y/Y to yank it literally.
Width budget: DUR is reserved at its real content width (≤7) and donates its space to TARGET on rows with no duration (SessionStart, Stop, Notification). In Browse, TARGET ≈ 37 cols so most real paths fit whole; in Drill it shrinks to ≈ 22, but every clipped value there is backed by the adjacent Inspector.
